###git是一种版本分发控制器，是一个管理代码用的软件。什么是版本控制系统？版本控制系统是一个实现不在原文件上直接修改而是在复制一份并在副本中进行修改的过程中进行控制功能的一个应用程序。

###而我们进行控制的文件都要提交的一个仓库（一个隐藏的文件夹），而这个仓库可以分为本地仓库（建在本地文件夹）和远程仓库（建在互联网sever内的文件夹）。

###版本控制系统分为2种：
一种是分布式版本控制系统（例如：git），他配有上述的两种仓库，提交文件时，我们会先提交到本地仓库，然后在联网条件下再提交到远程仓库。（Git按元数据方式将内容存储起来。）
另一种是集中式版本控制系统（典型的有svn),他只配有远程仓库，但是如果本地电脑充当远程服务器的话，远程仓库就在本地电脑上，而我们提交文件时也是直接提交到远程仓库上。（svn按文件存储内容）

###而Github和码云（Gitee)就是充当了一个远程仓库的身份，成为一个存放在外网server上的文件夹。

###Gitlab则是用于仓库管理系统的开源项目软件，以git作管理代码的工具的基础上搭建web服务。

##git的运作原理：
从工作区（working directory）[本地文件夹和文件]添加（git add）到缓存空间（staging area）[属于版本库虚拟部分，由于提交修改的文件不一定全部都提交到库中，所以就分这一部分空间用作缓存]提交（git commit）版本库（repository）[储存版本的地方]

##git commit命令：
git commit -m “message”是比较常见的用法，-m 参数表示可以直接输入后面的“message”，如果不加 -m参数，那么是不能直接输入message的，而是会调用一个编辑器一般是vim来让你输入这个message。

git commit -a -m “massage”加的-a参数可以将所有已跟踪文件中的执行修改或删除操作的文件都提交到本地仓库，即使它们没有经过git add添加到暂存区，但是新加的文件（即没有被git系统管理的文件）是不能被提交到本地仓库的。建议一般不要使用-a参数，正常的提交还是使用git add先将要改动的文件添加到暂存区，再用git commit 提交到本地版本库。

git commit --amend也叫追加提交，它可以在不增加一个新的commit-id的情况下将新修改的代码追加到前一次的commit-id中。

git commit --help 查看帮助，可以查看其他参数的效果。

##利用git，node，GitHub，hexo搭建一个博客：
首先，下载node.js和git资源的安装包并安装。
然后打开cmd命令行：
先通过node -v，git –version和npm -v查看node,npm和git是否安装成功以及版本代号
然后就是在本地安装hexo:
通过代码mkdir hexoBlog创建hexoBlog的文件夹，然后cd hexoBlog移动到hexoBlog文件夹中。接着npm install hexo -g开始安装hexo,安装完毕后通过hexo -v查看，接着通过hexo init将hexo初始化，接着npm install安装所需组件。然后hexo generate生成静态文件，接着hexo server 启动服务，如果你想要改端口号的话可通过hexo server -p +4位数端口号。然后我们就可以在本地浏览器访问hexo了。
接下来就将本地的hexo提交到github上，到github注册账号并创建库（ps:库名和用户名一样即可）注册成功后我们就可以打开gitbash窗口通过$git config –global user.name “你的用户名” 和$git config –global user.email”你注册的邮箱地址”来让GitHub知道谁上传东西。然后在gitbash窗口中通过$ls -al ~/.ssh（检查电脑是否有ssh），$ssh-keygen -t rsa -C”你注册的邮箱地址”（生成.ssh目录和里面的id_rsa私钥，id_rsa.pub公钥)以及$clip < ~/.ssh/id_rsa.pub(复制公钥）。
然后到GitHub的设置中给账户添加新的sshkey用以配置连接git。
然后在gitbash窗口中通过$ssh -T git@github.com来测试是否配置成功。
接下来到hexoBlog文件夹中的_config.yml中修改deploy值：
deploy:
tyep:git
responsitory:你创建的库的ssh路径。
branch：master
然后通过在cmd窗口代码npm install hexo-deployer-git -ave把文件交给GitHub。在此之前，可以先通过npm audit fix然后输入npm audit fix –force,再输入npm audit，然后再输入npm install lodash和eslint@^4.12.0等，安装这些扩展和依赖来让hexo deploy命令能够正常运行。
接着通过hexo generate（hexo g简写命令也是可以的)生成静态文件，然后再输入hexo deploy，博客便搭建完毕。

###git上传github库过程（gitbash窗口）：
$ mkdir xxxx 创建个文件夹
$ cd xxxx 进入这个文件夹
在文件夹中查看工作区状态：
$ echo “xxxxx” >> xxxx.txt
$ git status 查看状态
添加到缓存区并查看状态：
$ git add
$ git status
提交到库中并查看状态：
$ git commit -m “xxxx”
$ git status
其实可以将add和commit命令合并，但前提是文件早存在于库中，否则会报错：
$ git commmit -am “xxxxxxx”

###修改了的文件或者删除的文件并且是手动删除的工作区文件想撤销：
$ git status
$ git checkout – 撤销

###修改了或者删除并且手动删除的工作区文件并且提交到缓存区想撤销：
$ git status
$ git resset HEAD 取库的文件替换掉缓存区的文件
$ git checkout –

###git的命令：
$ git push origin master 推到远程仓库 origin代表远程仓库地址 master代表工作区或分支
$ git clone +ssh仓库地址 添加的管理员即可克隆远程仓库的文件到本地分支中
$ git pull origin master 拉远程仓库的文件到本地分支中
$ git log 查看项目日志
$ git log 查看项目文件的日志
$ git log . 查看本目录的日志
$ git log –pretty=oneline 让日志单行显示 还有second third four+line共4种版本切换
$ git reset –hard HEAD HEAD代表当前最新版本 加上^可切换到上一个版本，^数量代表往上切换多少个版本（这样切换不方便之后的切换）
$ git reset –hard 加上版本号 可直接到你想要到的版本，但只有这一个版本
$ git branch 查看所有分支
$ git branch +分支名字 创建生成一个分支
$ git checkout +分支名字 切换分支
$ git merge +分支名字 合并俩个分支，将添加的分支的文件添加到当前分支上
$ git branch -d +分支名字 删除分支
$ git remote 查看远程仓库别名
$ git remote -v 查看远程仓库地址
$ git remote remove +远程仓库名字 删除远程仓库别名
$ git remote add +远程仓库名字 添加远程仓库别名
